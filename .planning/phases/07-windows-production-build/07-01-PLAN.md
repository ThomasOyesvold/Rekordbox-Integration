<objective>
Bump Electron to fix node:sqlite regression, register custom audio protocol for production playback, and expose isProduction flag.

Purpose: Electron 37.2.0 has a node:sqlite binding bug (GitHub #47671) that crashes production builds. Additionally, production builds have webSecurity:true which blocks file:// audio URLs from the renderer. A custom `rbfa-audio://` protocol solves this by serving audio files through the main process with proper range request support for seeking.

Output: Updated package.json, main.js with protocol handler, preload.cjs with isProduction flag.
</objective>

<task type="auto">
  <name>Task 2: Register custom audio protocol and handler in main.js</name>
  <files>electron/main.js</files>
  <action>
CRITICAL PLACEMENT: `registerSchemesAsPrivileged` MUST be called at the MODULE TOP LEVEL, before `app.whenReady()`. If called inside whenReady, it fails silently.

1. At the TOP of electron/main.js (after imports, before any app.whenReady or other app code), add:

```javascript
import { protocol, net } from 'electron';
import { pathToFileURL } from 'node:url';

// MUST be at module top level -- before app.whenReady()
protocol.registerSchemesAsPrivileged([
  {
    scheme: 'rbfa-audio',
    privileges: {
      standard: true,
      secure: true,
      supportFetchAPI: true,
      stream: true,       // Required for audio seeking (range requests)
      corsEnabled: true
    }
  }
]);
```

Note: `protocol` and `net` may already be partially imported from 'electron'. Merge them into the existing import statement: `import { app, BrowserWindow, dialog, ipcMain, protocol, net } from 'electron';`. Also add `pathToFileURL` to the existing `import { fileURLToPath } from 'node:url';` line.

2. Inside the `app.whenReady().then(() => { ... })` block, BEFORE `createWindow()`, add the protocol handler:

```javascript
  // Handle rbfa-audio:// requests by serving local audio files
  protocol.handle('rbfa-audio', async (request) => {
    // URL format: rbfa-audio://local/<encoded-fs-path>
    const url = new URL(request.url);
    const encodedPath = url.pathname.startsWith('/')
      ? url.pathname.slice(1)  // remove leading /
      : url.pathname;
    const fsPath = decodeURIComponent(encodedPath);

    // Validate file extension for security
    const ext = path.extname(fsPath).toLowerCase();
    const allowedExts = ['.mp3', '.flac', '.wav', '.aac', '.m4a', '.ogg', '.wma', '.aiff', '.aif'];
    if (!allowedExts.includes(ext)) {
      return new Response('Forbidden file type', { status: 403 });
    }

    // Verify file exists
    if (!fsSync.existsSync(fsPath)) {
      return new Response('File not found', { status: 404 });
    }

    // Use net.fetch with pathToFileURL -- this handles Range headers automatically
    const fileUrl = pathToFileURL(fsPath).toString();
    return net.fetch(fileUrl, {
      headers: request.headers
    });
  });
```

3. Do NOT use `protocol.registerFileProtocol()` -- it was removed in Electron 35+. Use `protocol.handle()` only.

4. Do NOT touch any existing IPC handlers or the createWindow function's webSecurity setting (already correct: `webSecurity: !isDev`).
  </action>
