# Commit Notes

This folder stores one markdown file per commit to make debugging and rollback analysis easier.

## Naming

Use the pattern:

- `<short-hash>.md`

Example:

- `fd81dc7.md`

## Suggested Format

- Commit hash + title
- Date
- Why this change was made
- Files changed
- Validation done (tests/manual)
- Known risks

## Team Rule

When creating a new commit, add/update its matching file in this folder in the same working session.
